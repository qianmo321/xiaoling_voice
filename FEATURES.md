# 小灵语音助手 · 功能说明与测试指南

> 面向测试人员。技术部署见 `DEPLOY.md`。
> 测试地址：**https://192.168.2.80:8444**（须与服务器同内网；首次访问点"高级→继续前往"）

---

## 一、快速上手（3 步）

1. 浏览器打开测试地址 → 点 **「🎙️ 开始对话」** → 允许使用麦克风；
2. 喊 **「你好小灵」**（或"小灵小灵"）唤醒；
3. 直接说话提问。就这么简单。

> 💡 建议安静环境测试；用手机测须先点一次"开始对话"按钮（iOS 限制）。

## 二、界面说明

| 元素 | 说明 |
|------|------|
| 状态灯 + 文字 | ⚪灰=待机（要喊唤醒词）/ 🟢绿=空闲·请说话 / 🔵蓝闪=小灵说话中 / 🟠橙=重连中 |
| 对话气泡 | 蓝色=你说的（语音识别结果）；灰色=小灵的回答 |
| 灰色小字行 | 操作日志：`[唤醒]` `[联网搜索]` `[查知识库]` `[待机]` 等，展示它每一步在干什么 |
| 📜 日志:开/关 | 隐藏/显示操作日志（演示时可关掉） |
| 🌐 切换日本語 | 切换主语言（中↔日），小灵会口头确认 |
| 场景标签 | 当前所处场景（切换场景后出现） |
| 底部文字框 | 打字测试（不方便说话时用，功能与语音等价） |

## 三、功能详解

### 1. 唤醒与待机
| 行为 | 说明 |
|------|------|
| **唤醒** | 待机时喊「你好小灵」「小灵小灵」即唤醒。同音字（小玲/小凌/小林）、日语（シャオリン/こんにちは）、打招呼（你好）都可以 |
| **激活窗口** | 唤醒后可连续对话，**不用每句都喊小灵**；以"最后一方说完话"起算 **30 秒**无对话自动回待机 |
| **待机播报** | 进入待机时它会说："我已经有一段时间没有听到您的声音，现在进入待机状态啦…" |
| **口头待机** | 对话中说 **「进入待机状态」「休息吧」** → 它道别后立即待机 |
| **待机中** | 只认唤醒词，其它话一律不理（页面日志显示 `[待机忽略]`） |

**测试话术**：你好小灵 →（对话）→ 沉默30秒等播报 → 你好小灵 → 进入待机状态

### 2. 对话与打断 ⭐特色设计
| 行为 | 说明 |
|------|------|
| 正常对话 | 空闲时直接说，它听完自动回答 |
| **打断（需唤醒词）** | 它**正在说话**时，普通插话它会**当没听见继续说**；喊 **「你好小灵…」** 才会立刻停下听你的新问题 |
| 说完后 | 它说完后正常接话，**不需要**唤醒词 |

**测试话术**：问"详细介绍一下IST" →（它说长答案时）说"等一下"（预期：不理你）→ 说"你好小灵，换个话题"（预期：立刻停）

### 3. 场景切换 ⭐核心功能
三个场景，各有**独立的人设和知识库**：

| 场景 | 身份 | 知识范围 | 切换话术 |
|------|------|---------|---------|
| **机器人展厅**（默认） | IST 展厅导览讲解员 | IST/大连百易/百易东京公司资料、展板内容 | "切换到展厅模式" |
| **银座导览** | 东京银座商圈导览助手 | 百货/品牌店铺/美食/交通/历史文化 | "切换到银座模式 / 商场模式" |
| **清水寺导览** | 京都清水寺讲解员 | 历史/景点/参拜/门票交通 | "切换到清水寺模式 / 景点模式" |

日语切换：「ショッピングモールモードに切り替えて」「お寺モードにして」等。
切换成功它会以新身份口头确认，页面出现场景标签。

**测试话术**：切换到银座模式 → 银座有哪些百货公司? → 切换到清水寺模式 → 清水的舞台是什么? → 切换到展厅模式

### 4. 中日双语
| 行为 | 说明 |
|------|------|
| 回答语言 | **跟随你提问的语言**：中文问中文答、日语问日语答（不用切按钮） |
| 知识库双语 | 中文提问查中文库、日语提问查日文库（每个场景都有两套） |
| 🌐 按钮 | 切换"主语言"（影响它的默认语言、待机播报语言、识别偏好），每个浏览器独立 |

**测试话术**：（中文）介绍一下IST → （日语）ISTはどんな会社ですか → 点🌐切日语 → 听日语待机播报

### 5. 知识库问答（含展板介绍）
问到**当前场景**相关的内容，它会自动查本地知识库（日志显示 `[查知识库]`→`[知识库回答]`），**不编造**——库里没有就老实说不知道。

各场景可以问的示例：
- **展厅**："介绍一下IST" / "IST的社长是谁" / "法人番号是多少"（会逐位念）/ "大连百易是做什么的" / **"介绍一下你旁边这个展板"**（介绍完会主动提示可切换场景——只有问展板才带这句）
- **银座**："银座有哪些有名的百货公司" / "银座四丁目有什么" / "怎么去银座" / "有什么老字号"
- **清水寺**："门票多少钱" / "清水的舞台是什么" / "怎么去清水寺"

### 6. 天气查询与联网检索
**天气**（专用天气工具，日志显示 `[查天气]`→`[天气结果]`，数据来自气象服务，比搜网页更准）：
- 默认城市**跟随场景**：展厅=大连、银座导览=东京银座、清水寺导览=京都清水寺；
- 指明地点则查指定地点："东京天气怎么样"；
- **只报今天**的天气；明确问"明天/未来几天的天气"它才报未来的，不会主动播报。

**联网检索**：问**新闻、最新事实**等，它会自动联网搜索（日志显示 `[联网搜索]`→`[搜索完成]`），然后基于结果回答。

**测试话术**：今天天气怎么样（→大连）→ 切换到银座模式 → 今天天气怎么样（→东京银座）→ 明天天气呢（→报明天）→ 最近有什么AI新闻

> ⏱️ 知识库/联网问题回答前有 2~4 秒查询停顿，属正常。

### 7. 数字播报
编号类长数字（法人番号、电话号）会**逐位朗读**（"7、0、1、0…"），不会读成"七万亿…"。

## 四、"这不是 bug"清单（故意设计的不响应）

| 现象 | 原因（设计如此） |
|------|-----------------|
| 待机时说话不理你 | 没喊唤醒词 |
| 说"嗯""啊""好的"没反应 | 语气词过滤（日志 `[忽略语气词]`） |
| 它说话时插话没反应 | 打断需要唤醒词（日志 `[说话中忽略]`） |
| 屏幕出现"你: 谢谢观看"等没人说的话但它没回应 | 环境噪音被识别引擎"脑补"，已被拦截 |
| 极短的字/咳嗽声没反应 | 短音/杂音过滤 |
| 回答前停顿几秒 | 正在查知识库/联网（看日志） |

## 五、完整测试清单（照着过一遍）

```
□ 唤醒："你好小灵" → 有应答
□ 知识库："介绍一下IST" → 有 [查知识库] 日志、内容正确
□ 展板："介绍一下你旁边这个展板" → 介绍展板 + 结尾提示可切换场景
□ 天气："今天天气怎么样" → [查天气] 查大连、只报今天；切到银座模式再问 → 查东京银座
□ 联网："最近有什么AI新闻" → 有 [联网搜索] 日志
□ 打断：长回答中说"等一下"(不理) → "你好小灵换个话题"(立停)
□ 场景：切银座 → 问有什么百货公司 → 切清水寺 → 问门票 → 切回展厅
□ 日语：日语提问 → 日语回答；🌐切日语 → 日语待机播报
□ 待机：沉默30秒 → 自动播报进入待机；"进入待机状态" → 道别待机
□ 待机免打扰：待机中正常聊天(不带唤醒词) → 全程不理
□ 文字输入框：打字提问 → 正常语音回答
```

## 六、发现问题怎么反馈

请提供（缺一条排查就慢一倍）：
1. **页面截图**：包含对话气泡和灰色日志行（📜 保持开启）；
2. 你**实际说的话** vs 页面识别出的"你: …"是否一致；
3. 预期行为 vs 实际行为；
4. 时间点（方便管理员查服务器日志 `journalctl -u xiaoling`）。
